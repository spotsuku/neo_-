# NEO Design Bible

NEO ACADEMIA の全制作物（パンフレット・スライド・Web）で共有するデザイン基準。
新しい制作物を作るときは、まずこのファイルに準拠しているか確認すること。

---

## 1. ブランドの基本

| 項目 | 値 |
|------|----|
| ブランド名 | NEO ACADEMIA |
| トーン | 知的・誠実・前向き。過度な装飾を避け、余白で品位を出す |
| 判型（既定） | A4 縦（210 × 297 mm） |
| 背景 | 原則 **白**（#FFFFFF）。塗りつぶし背景は表紙・章扉のみ許可 |
| ページ番号 | パンフレットでは **なし**（仕様書で指定がある場合のみ付与） |

---

## 2. カラーパレット

### プライマリ

| 名称 | HEX | 用途 |
|------|-----|------|
| NEO Navy | `#102A71` | 見出し、ロゴ、キーカラー、図解の主線 |
| NEO Gold | `#D89B1D` | アクセント、強調、区切り線、CTA |

### ニュートラル（テキスト・地）

| 名称 | HEX | 用途 |
|------|-----|------|
| Ink | `#111827` | 本文テキスト |
| Sub | `#4B5563` | 補足・キャプション |
| Muted | `#9CA3AF` | 注記・弱いラベル |
| Line | `#E2E6EF` | 罫線・カード境界 |
| Paper | `#FFFFFF` | 背景 |
| Wash | `#F0F2F7` | カードのごく薄い地（多用しない） |

### 使用ルール
- 1ページに使う有彩色は **Navy + Gold の2色まで**。多色化しない。
- Gold は「全体の5%以内」を目安に。広い面積に Gold を敷かない。
- 文字色 on 白は Ink を基本。Navy 地の上は白文字（#FFFFFF）。
- 注意/警告など機能色が必要な場合のみ：成功 `#059669` / 警告 `#D89B1D` / 危険 `#DC2626`。

## 3. タイポグラフィ

### Web / HTML・SVG・PDF 生成時（既定）

| ロール | フォント | 指定 |
|--------|----------|------|
| 見出し（Display） | Noto Sans JP | weight 700–900 |
| 本文（Sans） | Noto Sans JP | weight 400–500 |
| 明朝が必要な箇所 | Noto Serif JP | weight 500–700 |

Google Fonts: `Noto+Sans+JP:wght@400;500;700;900` / `Noto+Serif+JP:wght@500;700`

### Office / PPTX 生成時（プロジェクト規約）
- serif: **MS明朝** / sans-serif: **Meiryo**（CLAUDE.md 準拠）
- pptxgenjs では絵文字を使わず番号ラベルを使用すること。

### サイズスケール（A4・96dpi 換算 px / pt 併記）
| ロール | px | pt | 行間 |
|--------|----|----|------|
| 表紙タイトル | 56 | 42 | 1.1 |
| 大見出し H1 | 32 | 24 | 1.2 |
| 中見出し H2 | 22 | 16.5 | 1.3 |
| 小見出し H3 | 16 | 12 | 1.35 |
| 本文 | 13 | 10 | 1.7 |
| キャプション | 10.5 | 8 | 1.5 |

## 4. レイアウト・グリッド（A4 縦）

- 外マージン：上下 18mm / 左右 16mm
- 本文エリア：178 × 261mm
- グリッド：12カラム、ガター 4mm
- 段落間スペース：本文1行ぶん（約 1.7em）
- 見出し上の余白 ≥ 見出し下の余白（呼吸を作る）

### ページ構成パターン
1. **表紙**：Navy ベタ or 白地 + 大タイトル + ロゴ + キービジュアル枠
2. **章扉**：Navy ベタ + 章番号（Gold）+ 章タイトル（白）
3. **本文1カラム**：見出し + リード + 本文 + 図版
4. **2カラム**：左テキスト / 右ビジュアル（5つの強みページ等）
5. **カードグリッド**：3〜4枚カードで要素を並列提示

---

## 5. コンポーネント

| 名称 | 仕様 |
|------|------|
| 見出しルール | H1/H2 の左に Gold の縦バー（幅 4px・高さ=行高）または下に Gold 罫 2px |
| カード | 角丸 8px、境界 1px `#E2E6EF`、内側 padding 14mm、影は使わない（印刷前提） |
| 画像枠 | 角丸 8px、プレースホルダは `#F0F2F7` 地 + 中央に Muted ラベル |
| 番号ラベル | Navy 円 or 角丸 + 白文字。強みページの「01〜05」に使用 |
| 区切り線 | Gold 2px（短め・装飾）/ Line 1px（機能的な区切り） |
| ロゴ位置 | 表紙＝中央 or 左上、本文＝各ページ右下 or なし |
| フッター | パンフは原則なし。社外向け資料はページ右下に小さく |

---

## 6. やってはいけないこと（NG）

- 3色以上の有彩色を1ページに使う
- Gold を広い面積のベタ塗りに使う
- 影・グラデーションの多用（印刷で破綻しやすい）
- 絵文字（特に PPTX）。代わりに番号・記号ラベル
- 中央寄せの長文本文（読みにくい。本文は左揃え）
- 解像度の低い画像の引き伸ばし

## 7. 制作フロー（推奨）

```
Claude Code（司令塔）
  ├─ copy_agent    : 構成・見出し・本文
  ├─ visual_agent  : 画像生成プロンプト（OpenAI / Canva）
  ├─ layout_agent  : HTML/SVG で A4 ページを生成
  └─ qa_agent      : 誤字・色・余白・整合性チェック
        ↓
  HTML/SVG → PDF/PNG 出力 → デザインツール（Canva/Figma）へ貼付 → 最終微調整
```

理由：複雑な図解はツール手操作より **コード生成（HTML/SVG）の方が再現性が高い**。
ゼロからツールでデザインさせず、本 Bible のトークンとマスターテンプレートを組み合わせる。

> 接続済みデザインツール：**Canva MCP**（`create-design-from-brand-template` 等）。
> Figma MCP は本セッション未接続。導入時は `claude plugin install figma@claude-plugins-official`。
