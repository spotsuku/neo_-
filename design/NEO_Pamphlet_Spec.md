# NEO パンフレット生成仕様書

Claude Code に一括指示するための「発注仕様」の標準フォーマット。
このファイルを埋めて渡せば、`copy → visual → layout → qa` の各エージェントが順に動く。

---

## 0. 一括指示テンプレート（コピーして使う）

```
NEO ACADEMIA パンフレットを更新してください。
- 対象     : <例: Canva版15ページをベースに再構成>
- 追加     : <例: P01〜P05 の「5つの強み」ページ>
- デザイン : NEO Design Bible に準拠 / 背景白 / Navy #102A71 / Gold #D89B1D
            / ページ番号なし / A4縦
- 出力     : 確認用PDF / 修正ログ / (デザインツールファイル)
- 地域版   : <例: 福岡版 / 熊本版>
```

---

## 1. 成果物の定義

| 項目 | 既定値 |
|------|--------|
| 判型 | A4 縦（210 × 297mm） |
| ページ数 | 可変（表紙 + 本文 + 裏表紙） |
| 出力形式 | ①HTML/SVG（編集元）②PDF（確認用）③PNG（QA用 150dpi）④デザインツール（Canva/Figma） |
| 命名規則 | `pamphlet_<area>_<yyyymmdd>.{html,pdf}` 例 `pamphlet_fukuoka_20260613.pdf` |
| 配置 | `design/output/` に出力（成果物はgit管理外でも可） |

---

## 2. 標準ページ構成（15ページ例）

| P | 種別 | 内容 |
|---|------|------|
| 表紙 | cover | タイトル / キービジュアル / ロゴ |
| 01 | message | 代表メッセージ・ミッション |
| 02 | overview | NEO ACADEMIA とは |
| 03–07 | strength | 5つの強み（番号ラベル 01〜05、2カラム） |
| 08 | program | プログラム / カリキュラム |
| 09 | flow | 受講の流れ（図解） |
| 10 | voice | 受講者の声（カードグリッド） |
| 11 | data | 実績・数値（KPIカード） |
| 12 | faq | よくある質問 |
| 13 | access | アクセス・拠点（地域版で差し替え） |
| 裏表紙 | back | 連絡先 / 申込導線 / ロゴ |

> 地域版は P13（access）と表紙キービジュアルのみ差し替えるのが基本。

---

## 3. エージェント処理パイプライン

```
発注仕様（本ファイル）
  │
  ├─[01_copy_agent]  → 構成・見出し・本文（trueな数値以外は捏造しない）
  ├─[02_visual_agent]→ 各ページの画像生成プロンプト（OpenAI/Canva）
  ├─[03_layout_agent]→ design/templates/neo_a4_master.html を複製して各ページ生成
  └─[04_qa_agent]    → 誤字・色・余白・ページ整合性チェック → 修正ログ
        ↓
   PDF/PNG 出力 → Canva/Figma へ取込 → 最終微調整
```

### 各段の入出力
| Agent | 入力 | 出力 |
|-------|------|------|
| copy | 発注仕様 + Bible | `copy.md`（ページ別テキスト） |
| visual | copy.md | `prompts.md`（画像プロンプト）+ 生成画像 |
| layout | copy.md + 画像 + master.html | `pamphlet_*.html` |
| qa | pamphlet_*.html | `qa_report.md`（修正ログ） |

---

## 4. QA チェックリスト（qa_agent が必ず確認）

- [ ] 誤字脱字・表記ゆれ（NEO ACADEMIA の表記統一）
- [ ] 有彩色は Navy + Gold の2色以内 / Gold は面積5%以内
- [ ] 背景は白（章扉・表紙の Navy ベタを除く）
- [ ] 外マージン 上下18mm・左右16mm を死守
- [ ] 見出し階層（H1>H2>H3）の整合
- [ ] 画像の解像度（150dpi 相当以上）・引き伸ばしなし
- [ ] ページ番号なし（仕様で指定がない限り）
- [ ] 数値・固有名詞の事実確認（捏造禁止）
- [ ] 印刷プレビューで改ページが崩れていない（@page）

---

## 5. 出力ツール連携

| ツール | 状態 | 用途 |
|--------|------|------|
| Canva MCP | **接続済（本セッション利用可）** | ブランドテンプレート→デザイン生成→PDFエクスポート |
| Figma MCP | 設定済 / 認証待ち | デザイン微調整（接続手順は README/Bible 参照） |
| HTML→PDF | 常時可 | ブラウザ印刷 or `soffice` / `wkhtmltopdf` |

> 推奨：まず HTML/SVG で版面を確定 → PDF/PNG 化 → デザインツールは「最後の微調整」に限定。
